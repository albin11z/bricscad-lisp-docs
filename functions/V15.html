  • Improvements and Additions for BLADE
    the "Syntax & Variables" dialog provides a nice improvement :
    for a selected file or DEFUN scope, a new list box shows all referenced Lisp files, which are necessary to run that selected file/defun;
    this allows to get a quick overview on mandatory Lisp file code which needs to be loaded along.

    Furthermore, a double-click on such a file entry in the new list box opens a new "Syntax & Variables" dialog instance, which then analyses that double-clicked Lisp file, and so forth.

    Debugging is now significantly faster :
    several improvements for the Debugger (memory management, data + code breakpoint handling) result in much better performance when Lisp code is processed under Debugger ... mainly for loop-based code, improvements are up to factor 5;
    DebugStep modes (SingleStep, StepOver) were fine-tuned and improved for a more consistent + predictable behaviour;
    CodeBreakPoints are no longer triggered multiple times for same line of code;

    Another important improvement :
    debugging pre-loaded Lisp files (loaded into BricsCAD outside BLADE & Debugger) is now possible;
    BLADE Debugger will automatically recognise - from StepInto debug step mode, from active Code-, Data-, SystemVariables-BreakPoints - when there is a Halt operation on such pre-loaded Lisp code, and will automatically + dynamically load that Lisp file code under Debugger, preserving all global variables from being overwritten, and not executing any load-time Lisp code (as the Lisp file was already loaded, to preserve application logic).
    That way, Debugging Lisp code is as simple as never before, virtually fool-proof :-)

    And finally, a number of visual glitches have been fixed
