PointGroup Filter Functions


All Civil3D PointGroup Filter related functions use the prefix : "civil:pgroup-filter-".

Each Civil3D PointGroup contains a number of filter pairs per filter parameter - each pair contains an "including" filter definition, and also an excluding filter definition;
all filter definitions are strings, specifying what to include resp. what to exclude.

Wildcards are supported for string-based properties (name, description, etc.), and range/interval based expressions for numerical parameters (height, position, elevation, etc.).
An empty, undefined filter is specified as an empty filter string "".

Examples :
"1,2,3"  "1-5,10"  i.e. for point numbers
">10.5"  ">1.0,<10.5"  i.e. for height, elevation, coordinate values
"Parking*"  i.e. for name, description

Important : if both "including filter" and "excluding filter" are provided as non-empty strings, the excluding filter has priority !
The PointGroup finally treats a specific Civil3D Point as member of the group, if any "including filter" matches the point parameters, and if not any "excluding filter" matches the the point parameters !
